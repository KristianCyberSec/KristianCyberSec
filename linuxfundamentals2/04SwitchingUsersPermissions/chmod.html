Using the chmod Command

The chmod command in Linux is used to change the access permissions of files and directories. It allows you to modify who can read, write, and execute a file.

Examples

chmod 755 script.sh - This command gives the owner of script.sh read, write, and execute permissions, while others receive only read and execute permissions.

Scenario 16: Changing Permissions of a Script

Task: You need to make the script backup.sh executable.

Command: chmod +x /home/user/backup.sh

Next Step: Use ls -l to confirm the script is now executable.

Permission Types

Linux file permissions are represented by three digits, where each digit corresponds to the permissions for the owner, group, and others, respectively. Each digit is a sum of the following values:

Read (r) = 4

Write (w) = 2

Execute (x) = 1

Here are some examples of common permission numbers:

755: This gives the owner read, write, and execute permissions (7), while the group and others get read and execute permissions (5). This is commonly used for executable scripts and applications.

644: The owner gets read and write permissions (6), while the group and others get read-only permissions (4). This is typically used for text files or configuration files that should not be modified by anyone other than the owner.

700: Only the owner gets full read, write, and execute permissions (7), while both the group and others have no permissions (0). This is often used for private scripts or sensitive files.

To summarize, the first digit represents the owner's permissions, the second digit represents the group's permissions, and the third digit represents the permissions for all other users.

For example:

777: Everyone (owner, group, others) has full read, write, and execute permissions.

755: The owner has full permissions, while the group and others can only read and execute.

600: Only the owner can read and write; no one else has access.
