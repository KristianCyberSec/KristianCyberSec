Understanding Output Redirection

Output redirection is a fundamental concept in Linux that uses the > and >> operators to direct the output of commands to files instead of the screen.

This capability is crucial for saving data, logging, and managing output in scripting.

Redirection Commands

  • > - Redirects the output of a command to a file, replacing the existing contents of the file.
  • >> - Appends the output of a command to the end of a file without removing existing content.

Example Usage

echo "Hello, world!" > file.txt - Redirects the string "Hello, world!" into a file named file.txt, overwriting its current contents.

echo "Add this to the file" >> file.txt - Adds the string "Add this to the file" to the end of file.txt without deleting the existing data.
