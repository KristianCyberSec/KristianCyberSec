Understanding Output Redirection

The > operator in Linux is used to redirect output to a file, overwriting its content.

This command is essential for saving command output directly to files, which is useful in scripts, logging, and file management.

Usage:

Use > to write command output to a file.

Example:

  • echo "Disk space is low" > /tmp/system_logs/alert.txt

    - This command creates a file alert.txt with the message "Disk space is low" inside the /tmp/system_logs directory.

Scenario 9: Saving System Alerts to a File

Task: You want to save a message about low disk space to an alert file.

Command: echo "Disk space is low on /dev/sda1" > /tmp/system_logs/alert.txt
