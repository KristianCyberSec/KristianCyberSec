Getting Started with Research

Research is a crucial skill for becoming a better hacker. You can’t know everything, so learning how to research effectively is vital. You will often use Google to stay up to date with current trends, new hacks, and to learn about specific concepts in depth.

For example, if you want to know what can be stored inside an image file, a quick search will reveal the concept of **steganography**, which involves hiding data within images. Further research might lead you to discover a tool called **Steghide**, which allows you to store and extract embedded data from images. By digging deeper, you’ll also find that Steghide can be installed using **apt**, a command-line tool for managing software packages on Ubuntu.

Not sure what apt is? A quick search will explain that it’s a command-line tool used for installing, upgrading, and managing software packages on Ubuntu. To install a package like Steghide, you’d simply use the command sudo apt-get install steghide, and you’re good to go.

Once you have Steghide installed, you can continue researching how to use it to extract hidden data from images. This process demonstrates the importance of research in hacking. You start with nothing, and through research, you gradually build up your knowledge. The ability to dive into unfamiliar topics and figure out solutions is essential to progressing in cybersecurity.

What I Learned

I learned how to break down big questions into smaller queries, gradually building an understanding. For example, researching **steganography** led me to tools like Steghide, and learning about **apt** helped me install packages efficiently. This allowed me to fully grasp how these tools work, from installation to actual use.

Another significant takeaway was understanding how to make search queries more efficient. Initially, I would search broadly, but with practice, I learned to narrow my queries, target specific results, and utilise advanced search operators to filter the most relevant content.

Challenges I Faced

The main challenge I faced was knowing where to start with vague questions. Learning to refine search queries and narrowing down results to find the most useful information was tough but rewarding. At times, I was overwhelmed by the volume of information online, and filtering through it for accurate, up-to-date, and relevant data was challenging. Balancing between general information and technical specifics was also a learning curve.

Another challenge was understanding unfamiliar terminology or tools. Sometimes research uncovered new concepts or tools that required further exploration, which made it a multi-layered process. While overwhelming at first, the process became smoother with practice.

Tips & Tricks

Here are some useful strategies that helped me improve my research skills:

  • **Break questions into smaller parts**: Instead of trying to answer broad questions, break them down into more manageable queries.
  • **Use diverse sources**: Rely on a combination of official documentation, blogs, forums, and Q&A sites like Stack Overflow.
  • **Follow a logical sequence**: Start with a general search to gain an overview, then explore specific tools, commands, or techniques as needed.

Resources

These are some of the resources I frequently used during my research:

  • Google: For general searches and quick answers.
  • Official Documentation: Always check the official documentation for tools and systems you are researching. For example, Ubuntu documentation was essential for learning about apt.
  • TryHackMe Modules: Helpful for hands-on learning and practicing specific hacking techniques.

Research Methodology

Here’s a simple methodology I follow for effective research:

  1. Start with a question: Identify the problem or topic you need to explore.
  2. Build a basic understanding: Conduct general research to get an overview of the subject.
  3. Explore more advanced topics as needed: Once you have the basics, dive deeper into specifics, whether it’s about a tool, technique, or concept.
